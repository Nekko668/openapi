创建部门 ​

描述： 创建一个部门
调用方式： POST
接口请求： https://[server:port]/openapi/v1/departments/create

请求参数

以下请求参数列表仅列出了接口请求参数中Data参数里的JSON 数据参数，HTTP 请求公共参数请参见签名验证章节的 公共参数说明。

参数名必选类型说明
department_nameYString部门名称
department_parent_idYString部门的父ID
department_serialNString部门序列
department_order_indexNInt部门排序

返回参数

以下返回参数列表仅列出了返回参数中Data里的JSON 数据参数，公共返回参数请参见签名验证章节的 公共参数说明。

参数名类型说明
department _idInt用户ID
department _nameString用户帐号

请求示例

json
{ 
    "department_name":"test",
    "department_parent_id":2
}

返回示例

json
{
    "code": "200",
    "message": "成功",
    "data": {
      "department_id": 47,
      "department_name": "test1"
    },
    "status": "success",
    "timestamp": 1654742281016
}

获取部门列表 ​

描述： 获取所有部门列表
调用方式： POST
接口请求： https://[server:port]/openapi/v1/departments/list

请求参数

以下请求参数列表仅列出了接口请求参数中Data参数里的JSON 数据参数，HTTP 请求公共参数请参见签名验证章节的 公共参数说明。

参数名必选类型说明
department_id
department_serial		YInt
Stirng		部门id和序列号二选一

返回参数

以下返回参数列表仅列出了返回参数中Data里的JSON 数据参数，公共返回参数请参见签名验证章节的 公共参数说明。

参数名类型说明
arr_departmentarray部门信息的数组
+ department_nameString部门名称
+ department_parent_idInt部门父ID
+ department_parent_serialString部门父序列(第三方同步时的唯一标识)
+ department_idInt部门ID
+ department_serialString部门序列
+ department_gradeInt部门级别
+ department_order_indexInt部门排序序号
+ children_departmentarray子部门

请求示例

json
{
    "department_id":1
}

返回示例

json
{
  "error_code": 200,
  "msg": "success",
  "timestamp": 1686110318184,
  "data": {
    "arr_department": [
      {
        "department_name": "test",
        "department_id": 12,
        "department_parent_id": 1,
        "department_serial": "",
        "department_parent_serial": "",
        "department_grade": 2,
        "department_order_index": 3,
        "children_department": [
          {}
        ]
      }
    ]
  }
}
