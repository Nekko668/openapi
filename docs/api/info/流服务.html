请求设备/会议视频流 ​

描述： 获取设备流的信息
调用方式： POST
接口请求： https://[server:port]/openapi/v1/stream

请求参数

以下请求参数列表仅列出了接口请求参数中Data参数里的JSON 数据参数，HTTP 请求公共参数请参见签名验证章节的 公共参数说明。

参数名必选类型说明
device_id
device_sn
room_id
username		YInt设备ID
设备SN
会议室ID
用户帐号
channelNInt多路视频源时，指定通道号

返回参数

以下返回参数列表仅列出了返回参数中Data里的JSON 数据参数，公共返回参数请参见签名验证章节的 公共参数说明。

参数名类型说明
rtmp_urlStringrtmp流地址
hls_urlStringhls流地址
http_flv_urlStringhttp-flv流地址
webrtc_urlStringwebrtc流地址
tokenString访问流的token
statusInt1直播推流成功 2设备正在响应 3连接流媒体服务器失败 4发布流失败
urlsArray流媒体有多IP（内外网的），所有的流地址

请求示例

json
{
  "device_id": 1
}
// 示例参数
param = {
  "device_id": 1
}

// 将param进行Base64编码
param_base64 = encodeURIComponent(base64(param))

// 对参数进行签名
// 对拼接参数进行MD5
sign_md5 = MD5(access_token + param_base64 + timestamp)

// 用appkey对MD5的数据进行HmacSHA1
sign_sha = encodeURIComponent(base64(HmacSHA1(sign_md5, appkey)))

POST 
https://[server:port]/openapi/v1/stream
access_token=xxxx&data=param_base64&timestamp=1529223702&sign=sign_sha
