创建用户 ​

描述： 创建一个用户
调用方式： POST/GET
接口请求： https://[server:port]/openapi/v1/users/create

请求参数

以下请求参数列表仅列出了接口请求参数中Data参数里的JSON 数据参数，HTTP 请求公共参数请参见签名验证章节的 公共参数说明。

参数名必选类型说明
user_nameYString用户帐号
user_passwordYString用户密码（MD5 32位大写加密）
user_nicknameNString用户昵称
user_true_nameNString用户姓名
user_phoneNString用户手机号
user_emailNString用户邮箱
user_serialNString用户序列（第三方同步时的唯一标识）
department_ids
department_serials		YString部门ID/部门序列 二选一 （多个用","分隔）
sexNInt性别（0未知 1女 2男）
addrNString地址
user_typeYInt用户类型（4普通用户，67视频调度用户）
user_order_indexNInt用户排序
is_validNInt是否启用（0否 1是，不传默认为1）

返回参数

以下返回参数列表仅列出了返回参数中Data里的JSON 数据参数，公共返回参数请参见签名验证章节的 公共参数说明。

参数名类型说
arr_userJSON Object用户信息的对象

用户信息

参数名类型说明
+ user_nameString用户帐号
+ user_idInt用户ID
+ error_codeInt错误号

请求示例

json
{
  "user_name": "test3",
  "user_password": "Aa123..",
  "user_nickname": "昵称1",
  "user_true_name": "真实名1",
  "user_phone": "11111111111",
  "user_email": "11111111111@qq.com",
  "user_serial": "11111111111",
  "department_ids": [
    1,
    13
  ],
  "user_type": 4
}
// 示例参数
param = {
  "user_name": "test3",
  "user_password": "Aa123..",
  "user_nickname": "昵称1",
  "user_true_name": "真实名1",
  "user_phone": "11111111111",
  "user_email": "11111111111@qq.com",
  "user_serial": "11111111111",
  "department_ids": [
    1,
    13
  ],
  "user_type": 4
}

// 将param进行Base64编码
param_base64 = encodeURIComponent(base64(param))

// 对参数进行签名
// 对拼接参数进行MD5
sign_md5 = MD5(access_token + param_base64 + timestamp)

// 用appkey对MD5的数据进行HmacSHA1
sign_sha = encodeURIComponent(base64(HmacSHA1(sign_md5, appkey)))

POST 
https://[server:port]/openapi/v1/users/create
access_token=xxxx&data=param_base64&timestamp=1529223702&sign=sign_sha

返回示例

json
{
  "code": "200",
  "message": "成功",
  "status": "success",
  "timestamp": 1654663292204,
  "data": {
    "arr_user": {
      "user_name": "test1",
      "user_id": 132,
      "error_code": "0"
    }
  }
}

error_code 请参见 错误号 (error_code) 定义
